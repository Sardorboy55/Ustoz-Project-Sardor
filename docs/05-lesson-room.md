# 05 — Урок внутри платформы

## 5.1 Видеокомната

**Доступ:** кнопка «Войти в урок» активируется за 10 минут до `start_at`. Edge Function `agora-token` проверяет, что `auth.uid()` — участник брони и сейчас внутри окна `[start−10мин, start+duration+30мин]`, выдаёт RTC-токен (канал = `booking_id`, TTL = до конца окна).

**Интерфейс комнаты (Flutter / Web):**
- Видео собеседника на весь экран, своё — плавающее окно (перетаскиваемое).
- Панель: микрофон вкл/выкл · камера вкл/выкл · переключить камеру · демонстрация экрана (teacher, Web обязательно; mobile — если позволит SDK) · доска · чат · файлы · таймер урока · выйти.
- Индикаторы: качество сети, «собеседник подключается…», реконнект при обрыве (авто, до 60 сек).
- Вход участника пишет `lessons.student_joined / teacher_joined = true`, `started_at` при первом входе обоих.
- За 5 минут до конца — мягкое уведомление в комнате. По `start_at + duration + 30 мин` токены умирают.
- **No-show:** если через 15 мин после старта одна из сторон не вошла — баннер «Сообщить о неявке» у второй стороны → статус `no_show_*` (правила в docs/01).

**Завершение:** любой участник жмёт «Завершить» (подтверждение) или авто-завершение cron'ом. Статус `completed` → ученику экран оценки (звёзды + текст, можно пропустить) → XP.

**VideoProvider (абстракция, критично для миграции на LiveKit):**

```ts
interface VideoProvider {
  // сервер (Edge Function)
  createToken(bookingId: string, userId: string, role: 'student'|'teacher'): Promise<{token: string, channel: string, expiresAt: string}>
}
// клиент (Flutter): абстрактный класс VideoSession
abstract class VideoSession {
  Future<void> join(String channel, String token);
  Future<void> leave();
  void toggleMic(bool on); void toggleCam(bool on); void switchCamera();
  Future<void> startScreenShare(); Future<void> stopScreenShare();
  Stream<VideoEvent> get events; // joined, left, networkQuality, error
}
```

Реализация №1 — `AgoraVideoSession` (agora_rtc_engine). Реализация №2 (фаза 7+) — `LiveKitVideoSession`. UI комнаты не знает про конкретный SDK.

## 5.2 Интерактивная доска

Собственная доска на **Supabase Realtime** (без платных whiteboard-SDK):

- Канал Realtime `whiteboard:{booking_id}`. Каждый штрих — insert в `whiteboard_strokes` + broadcast; вторая сторона дорисовывает по событиям. При входе — загрузка истории штрихов (replay).
- Инструменты: перо (3 толщины, 6 цветов), ластик, текст, прямая линия, прямоугольник/круг, «очистить всё» (event `clear`), undo своего последнего штриха.
- Вставка изображения: загрузка в Storage → event `{type:'image', url, x, y, w, h}` (учитель показывает упражнение/страницу учебника).
- Полотно бесконечное со скроллом, фон — клетка/чистый.
- Открывается поверх видео (видео сворачивается в маленькие окна сверху).
- Экспорт доски в PNG по окончании → прикрепляется к карточке прошедшего урока.

Точки штриха сэмплируются (каждые ~16мс, упрощение Дугласа–Пекера) — payload лёгкий, Realtime тянет.

## 5.3 Чат

- Постоянный чат «ученик–преподаватель» (один на пару), доступен до/после/во время урока; во время урока — та же лента в панели комнаты.
- Текст + файлы (pdf, doc, изображения, аудио до 20 МБ) через Storage bucket `chat-files` (signed URLs).
- **Маскировка контактов** (insert-триггер `mask_contacts`): телефоны (+998…, 9 цифр подряд и форматы), `t.me/…`, `@handle`, `instagram.com/…`, прочие URL вне доменов платформы → `[скрыто]`, флаг `body_was_masked`. Сообщения с 3+ срабатываниями попадают в очередь модерации админки.
- Realtime-доставка, статусы «отправлено/прочитано», typing-индикатор, push на новое сообщение (если чат не открыт).

## 5.4 Файлы урока

Вкладка «Материалы» в карточке урока: всё, чем обменялись в чате этого периода + загруженное в комнате + PNG доски. Хранится 12 месяцев.

## 5.5 Домашние задания и тесты

**Создание (преподаватель):** заголовок, описание, вложения, дедлайн, опционально тест-конструктор: N вопросов × 2–5 вариантов, отметка правильного. Привязка к ученику (и опц. к уроку).

**Выполнение (ученик):** push «Новая домашка» → экран задания → ответ текстом/файлами и/или прохождение теста (автопроверка: % правильных, мгновенный результат) → submit.

**Проверка:** преподаватель ставит оценку 1–5 + комментарий → push ученику → XP (+20 за сдачу, +10 за тест ≥80%).

## 5.6 Геймификация

- XP-события по правилам из `app_settings.xp_rules` (см. docs/01). Начисление — функцией `grant_xp(user_id, kind, ref_id)` (идемпотентно по `(kind, ref_id)`).
- Стрик: любой день с уроком или сданной домашкой. Пропуск дня: если есть заморозка — тратится автоматически, иначе стрик = 0. Cron в 00:05 Asia/Tashkent.
- Уровни по порогам XP (`level_thresholds`), анимация level-up в приложении.
- Экран «Прогресс»: уровень+XP-бар, стрик с календарём активности, счётчики уроков/домашек по предметам.
