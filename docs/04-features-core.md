# 04 — Ядро платформы: фичи и экраны

## 4.1 Авторизация (телефон + SMS)

Поток: ввод номера `+998 XX XXX XX XX` → Edge Function `sms-hook` шлёт 6-значный код через Eskiz → ввод кода (автоподстановка из SMS на Android) → если новый пользователь: имя + выбор интересов → главная.

- Rate limit: 3 SMS / 10 мин, 5 попыток кода, код живёт 5 мин.
- Сессия Supabase JWT, refresh автоматический. Логаут — отзыв сессии + удаление FCM-токена.
- «Стать преподавателем» — отдельный онбординг из профиля (анкета → `is_teacher=true`, создаётся `teacher_profiles`, `wallets`).

## 4.2 Каталог и поиск

- Главная (ученик): персональные подборки по выбранным интересам, «Популярные преподаватели», «С бесплатным пробным», категории плиткой.
- Каталог: список карточек (фото, имя, headline, предметы, цена/60мин «от», рейтинг ★, бейджи PRO/Verified, кнопка «Пробный»).
- Фильтры: категория → предмет, цена min/max, рейтинг ≥4, язык преподавания (uz/ru/en), бесплатный пробный, доступен сегодня/на этой неделе.
- Сортировка: **рекомендуемые** (формула: `search_boost + rating_avg*10 + ln(lessons_done+1)*5`, забаненные и moderation_flag исключены), цена ↑↓, рейтинг.
- Поиск текстом: по имени/headline/предметам (pg_trgm), uz+ru.
- Профиль преподавателя: фото, видео-презентация (90 сек, плеер), headline, био, предметы с ценами 30/60/90, скидки на пакеты, рейтинг + отзывы (пагинация), календарь свободных слотов на 14 дней, кнопки «Бесплатный пробный» / «Забронировать» / «Написать».

## 4.3 Расписание и бронирование

- Преподаватель: редактор недельной сетки (тапами по 30-мин ячейкам), исключения «закрыть дату/период».
- Ученик при бронировании: предмет → длительность (30/60/90; 20 — только бесплатный пробный) → календарь свободных слотов (`get_free_slots`) → подтверждение цены → оплата.
- `booking-create` (Edge Function): проверяет слот ещё свободен, лимит FREE-преподавателя (10 уроков/мес — иначе ошибка `TEACHER_LIMIT`, ученику показываем «слот недоступен», преподавателю — paywall), создаёт `pending_payment` (TTL 15 мин), возвращает checkout-URL выбранной платёжки **или** списывает пакет/баланс мгновенно.
- Источники оплаты на чекауте: Payme / Click / Uzum / **баланс ученика** (если хватает) / **пакет** (если есть подходящий по предмету и длительности).
- После `paid`: push обоим, событие в «Мои уроки», план напоминаний (push −24ч, push −1ч, SMS −1ч).
- Отмена/перенос: по политике из `docs/01-product.md`; кнопки в карточке урока; возвраты — на `student_balance` через `student_balance_refund`.

## 4.4 Пакеты

- Покупка с профиля преподавателя: выбор предмета → длительность → 5/10/20 → цена со скидкой → оплата → запись в `student_packages`.
- При бронировании пакетным способом: `lessons_left -= 1` (в той же транзакции, что и бронь). Отмена ≥12ч возвращает кредит в пакет.
- Срок 6 мес; за 14 дней до сгорания — уведомление.

## 4.5 Кошелёк преподавателя и выплаты

- Зачисление: cron `lesson-complete-cron` через 24ч после `completed` → `wallet_credit_lesson` (+сумма, минус эквайринг ~2% — параметр `acquiring_pct`).
- Экран «Кошелёк»: баланс / заморожено, история транзакций, кнопка «Вывести».
- Заявка: сумма (мин. 50 000 UZS — настройка) + номер карты Humo/Uzcard → `payout-request` замораживает сумму → админ в панели «Выплатил» (вручную переводит) → `paid`, разморозка-списание; «Отклонил» → возврат на баланс + причина.
- Уведомления о смене статуса.

## 4.6 Подписка PRO

- Paywall-триггеры: достигнут лимит 10 уроков; попытка добавить 2-й предмет; экран «Тарифы» в кабинете.
- Оплата: `subscription-pay` → checkout платёжки → webhook → `tier='pro'`, `tier_expires_at += 1 мес`, `search_boost=100`.
- За 3 дня до окончания — push+SMS напоминание. По истечении: `tier='free'`, boost=0, лишние предметы деактивируются (не удаляются).
